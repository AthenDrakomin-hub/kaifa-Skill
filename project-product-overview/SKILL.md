# 产品概述生成

## 任务目标
- 本Skill用于：将项目描述升级为正式的产品文档
- 能力包含：产品愿景定义、核心价值提炼、系统模块划分、验收标准制定
- 触发条件：已完成需求澄清，需要生成结构化的产品规格

## 前置准备
- 必须存在 `specs/项目描述.md`
- 工作目录：项目根目录

## 操作步骤

### 1. 读取项目描述
智能体扮演首席产品官角色，首先读取需求澄清的成果：
```
读取 specs/项目描述.md
```

### 2. 信息完整性检查
检查项目描述是否包含以下关键信息：
- 项目愿景是否清晰
- 核心价值是否明确
- 目标用户是否具体
- 成功标准是否可量化

### 3. 引导式补充
对于缺失或模糊的信息，通过具体提问补充：

**核心价值澄清**：
```
你的博客和市面上其他博客的最大区别是什么？

A：主打 Markdown 写作体验，像 Typora 一样流畅
B：主打 SEO 和流量获取，帮你把文章推出去
C：主打极简部署，一键上线零运维
或者你有其他想法？
```

**验收标准量化**：
```
第一版上线的标准是什么？我建议至少定两条：

功能层面：比如"能发布文章、能按标签分类、有基础的阅读统计"
性能层面：比如"首页加载 < 2 秒"
```

### 4. 生成产品概述文档
生成完整的产品概述文档：

```markdown
# 产品概述

## 1. 项目愿景
[一句话描述项目的长期目标]

## 2. 核心价值
[描述产品的独特价值和竞争优势]

## 3. 目标用户
### 主要用户群体
[描述主要用户]

### 使用场景
[描述典型使用场景]

## 4. 系统模块
### 核心板块
1. [模块1名称]：[模块描述]
2. [模块2名称]：[模块描述]
3. [模块3名称]：[模块描述]

### 模块间关系
[描述模块之间的交互关系]

## 5. 功能清单
### MVP功能（第一版必须）
- [ ] [功能1]
- [ ] [功能2]
- [ ] [功能3]

### 完整版功能
- [ ] [功能4]
- [ ] [功能5]

### 增强版功能
- [ ] [功能6]
- [ ] [功能7]

## 6. 验收标准
### 功能验收
- [ ] [标准1]
- [ ] [标准2]

### 性能验收
- [ ] [标准1]
- [ ] [标准2]

### 用户体验验收
- [ ] [标准1]
- [ ] [标准2]

## 7. 非功能性需求
- 安全性要求
- 可维护性要求
- 可扩展性要求
```

### 5. 保存文档
将生成的产品概述保存到 `specs/产品概述.md`

## 输出格式
- 文件路径：`specs/产品概述.md`
- 文档格式：结构化的产品规格文档
- 内容要求：具体、可量化、可验证

## 注意事项

### 边界守卫规则
- **当前阶段只能做**：产品文档生成和产品层面的问题讨论
- **禁止行为**：不涉及技术选型、不涉及代码实现、不涉及数据库设计
- 如果用户问技术问题，提醒："我们现在在定义产品应该长什么样，技术实现的事后面会专门规划"

### 项目上下文协议
- 执行前必须读取 `specs/项目描述.md`
- 如果文档不存在，提示用户先执行需求澄清

### 提问技巧
- 每个问题都要给出选项和解释
- 选项要具体，避免模糊的描述
- 给出推荐选项并说明理由

