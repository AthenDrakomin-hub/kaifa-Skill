# 项目初始化

## 任务目标
- 本Skill用于：根据规划文档创建项目骨架和配置文件
- 能力包含：目录创建、依赖安装、配置生成、Git初始化
- 触发条件：已完成所有规划文档，准备开始实际开发

## 前置准备
- 必须存在 `specs/技术栈.md`
- 必须存在 `specs/项目结构.md`
- 必须存在 `specs/开发规范.md`
- 工作目录：项目根目录（建议空目录）

## 操作步骤

### 1. 环境检查
智能体扮演高级工程师角色，首先检查环境和文档：

**检查清单**：
```
✓ specs/技术栈.md … 已就绪
✓ specs/项目结构.md … 已就绪
✓ specs/开发规范.md … 已就绪
✓ 当前目录状态 … [空目录/已有文件]
✓ Node.js版本 … [版本号]
✓ 包管理器 … [npm/yarn/pnpm]
```

### 2. 执行初始化
严格按照文档执行初始化：

**初始化流程**：
```
开始执行项目初始化。

环境检查
[显示环境检查结果]

执行中…

1. 初始化项目
   - 执行 npm init -y … 完成
   - 或使用指定包管理器初始化

2. 安装核心依赖
   - 读取技术栈文档中的依赖列表
   - 安装生产依赖 … 完成
   - 安装开发依赖 … 完成

3. 创建目录结构
   - 读取项目结构文档
   - 创建所有目录（共 X 个目录）… 完成

4. 生成配置文件
   - .gitignore … 完成
   - .env.example … 完成
   - tsconfig.json … 完成
   - [其他配置文件] … 完成

5. 初始化Git仓库
   - git init … 完成
   - 首次提交 … 完成

初始化完成。
```

### 4. 保存初始化记录
将初始化记录保存到 `docs/开发记录/初始化记录.md`

### 5. 输出完成报告
向用户展示初始化结果：
```
项目初始化已完成！

已创建：
- X 个目录
- X 个配置文件
- X 个依赖包

下一步建议：
请按照 specs/开发路线图.md 中的 Phase 1.1 开始开发。
```

## 输出格式
- 项目骨架：完整的目录结构和配置文件
- 初始化记录：`docs/开发记录/初始化记录.md`
- 首次Git提交：包含所有初始化文件

## 注意事项

### 边界守卫规则
- **当前阶段只能做**：项目骨架创建和配置文件生成
- **禁止行为**：不涉及业务代码编写、不随意添加文档未定义的内容
- **严格遵循**：只创建文档中定义的目录和配置

### 项目上下文协议
- 执行前必须读取 `specs/技术栈.md`、`specs/项目结构.md`、`specs/开发规范.md`
- 严格按照文档执行，不自作主张

### 安全检查
- 检查目标目录是否已有文件
- 如果已有文件，提示用户确认是否继续
- 不覆盖已有文件（除非明确指定）

### 处理已有项目
如果项目目录非空：
```
检测到当前目录已有文件：
- [文件列表]

继续初始化将会：
- 保留已有文件
- 只创建不存在的目录和文件
- 更新 package.json（如需要）

是否继续？
```

## 异常处理

### 环境不满足
```
环境检查失败：
✗ Node.js 未安装
✗ npm 版本过低（需要 >= 16.0.0）

请先安装或更新 Node.js 环境。
```

### 文档缺失
```
文档检查失败：
✗ specs/技术栈.md 不存在
✗ specs/项目结构.md 不存在

请先完成技术选型和结构设计。
```

### 依赖安装失败
```
依赖安装失败：
✗ [包名] 安装失败

可能原因：
- 网络问题，请检查网络连接
- npm源问题，建议切换到淘宝源
- 包名或版本不存在

建议操作：
1. 检查网络连接
2. 尝试切换npm源：npm config set registry https://registry.npmmirror.com
3. 手动安装失败依赖
```
